# UI / UX Spec

## Design intent
Trustay should feel:
- calm
- clear
- useful
- trustworthy
- not cluttered

Avoid “travel app glamour” styling.
Favor utility and readability.

## Visual style
- clean spacing
- muted palette
- strong typography hierarchy
- simple cards
- no unnecessary animations
- no dashboard complexity

## Primary pages

### Homepage
Sections:
1. Hero
2. Search input
3. How it works
4. Why Trustay
5. Light pricing teaser

Hero copy direction:
- headline: "Choose your next remote-work base with confidence."
- subcopy: "See where to stay, work, train, and eat without losing days to research."

### City page
Layout order:
1. City hero summary
2. Recommended area card
3. Routine summary card
4. Work spots section
5. Coworking backup section
6. Training section
7. Food / coffee section
8. Paywall card
9. Methodology / honesty note

## Paywall placement
Do not block all value immediately.
Users should see enough to understand what the product does.
The paywall should appear after meaningful free value, not before.

## Place cards
Each place card should show:
- place name
- category
- rating and review count if available
- distance from base area
- confidence badges
- best-for tags
- one-line explanation

## Badge design
Badges must be compact and readable.
Examples:
- Work fit: Medium
- Noise risk: Unknown
- Wi‑Fi: Not verified

## Copy tone
- direct
- calm
- non-hype
- honest

Avoid exaggerated copy.
